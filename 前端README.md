由于改成了**横屏（Landscape）**，我们要对之前的空间布局做一次重构，彻底拥抱大屏的宽视野。以下是为您量身定制的前端进阶规划：

### 🎨 一、 核心视觉语言与 Tailwind 规范 (Design System)

要实现 Apple Fitness 那种高级的“磨砂毛玻璃 + 多彩圆环”质感，我们需要在 Tailwind 中建立一套专属的设计规范：

* **深色底座 (Deep Background)**：纯黑 `#000000` 或极深灰 `#0C0C0E`，确保设备的边框与屏幕融为一体。
* **毛玻璃面板 (Glassmorphism)**：大量使用 Tailwind 的 `backdrop-blur-xl` + `bg-white/10`（白底10%透明度） + `border border-white/20`。这种材质会作为所有卡片、侧边栏、弹窗的基础容器。
* **多彩荧光强调色 (Neon Accents)**：
* 运动活力：Apple 标志性的荧光绿 (Lime/Emerald) 和 亮橙色 (Orange)。
* AI 元素：渐变紫/深空蓝 (Fuchsia to Blue)，用于表达大模型在工作。


* **超大排版 (Macro Typography)**：由于是体测机，用户可能在一米外站立观看。字号要成倍放大，使用粗体（`font-bold` 或 `font-black`）配合紧凑的字间距（`tracking-tighter`）展示核心数字（如次数、心率）。

### 📐 二、 全局架构重构 (横屏特化)

横屏状态下，底部的导航栏会显得突兀且浪费垂直空间（影响视频流展示）。我们改为 **“左侧悬浮边栏 (Side Navigation) + 右侧大内容区”** 的布局。

* **全局左侧边栏 (Sidebar)**：
* **宽度**：约占据屏幕左侧 15%-20%。背景为极度模糊的毛玻璃。
* **内容**：顶部显示全局时间/网络状态图标；中间是 3 个大号图标+文字的 Tab（主页、运动、我）；底部预留给“AI语音小助手”的常驻入口。


* **右侧内容区 (Main Content)**：
* 占据剩余 80%+ 空间。页面切换时，使用简单的透明度淡入淡出（`Fade-transition`）或水平轻微滑动。

### 🔊 三、 AI 语音模块化设计 (前端主导)

既然你接管了语音的播放控制，前端的可玩性就大大增加了。我们可以设计一个全局单例组件 `<AIVoiceAssistant />`，通过 Pinia 状态管理来控制它的展现。

* **UI 形态**：
* 待机时：侧边栏底部一个小型的彩色渐变光球。
* 发声时：展开成一个**动态声波可视化组件 (Soundwave Bar)**。你可以利用 Web Audio API 读取播放的音频流数据，或者简单点，根据播放状态做一个 CSS 驱动的动态高度柱状图/光晕呼吸效果。


* **交互逻辑**：前端调用后端拿到 TTS 音频 URL 或 Blob 后，利用 `new Audio(url).play()` 播放。在 `.play()` 触发时改变状态为 `isPlaying: true`，UI 上呼出漂亮的 OSD 字幕框和跳动的声波。

### 🖥️ 四、 核心界面横屏排版设计

#### 1. 待机与登录界面 (Standby / Login)

* **背景**：全屏暗色调。
* **视觉中心**：屏幕正中央是一个巨大的**正圆形摄像头画面**（配合 `rounded-full` 和 `object-cover`）。圆的边缘加上一圈 Apple 风格的彩色渐变进度条动画，表示“人脸扫描中”。
* **信息层**：
* 圆环上方：大字体的高级感欢迎语，例如“靠近即可唤醒”。
* 圆环下方：一个毛玻璃质感的胶囊按钮「账号密码登录」。点击后，圆形摄像头缩小并平移至左侧，右侧滑出毛玻璃质感的数字全键盘和密码输入框。



#### 2. 主界面 (Home Dashboard)

* **左侧**：常驻毛玻璃侧边栏。
* **右侧布局 (CSS Grid 组合)**：
* **左半部分（个人状态卡片）**：一个巨大的圆环图表组件（类似 Apple Watch 的三大圆环，这里可以展示“今日负荷”、“计划进度”等），圆环中间用大号字体显示当天的“AI 综合评分”。
* **右半部分（行动区）**：上下排列两个长条形的毛玻璃按钮。
* 上（主按钮）：**「启动今日计划」**，带有动态流光边框效果。
* 下（次按钮）：**「自由训练」**，稍微透明一些。





#### 3. 自主训练设置页 (Free Training Setup)

* **双栏平铺**：
* **中列（选动作）**：一个支持垂直滚动的网格（Grid），每个动作是一个正方形卡片（比如深蹲、俯卧撑），选中时加上高亮的荧光绿边框。
* **右列（调参数）**：iOS 风格的 **3D 滚轮 (Picker)**。因为是横屏，空间充足，并排两个大滚轮：左边选“组数”，右边选“次数”。最下方是一个宽大的「开始训练」按钮。



#### 4. 训练执行界面 (Active Training - 沉浸式)

* **布局转变**：进入此页面，左侧边栏彻底隐藏，进入真全屏。
* **底层**：全屏的 WebRTC 摄像头画面，画面经过轻微的暗角处理（Vignette），让前方的 UI 更加清晰。
* **上层 UI (全部采用毛玻璃卡片悬浮)**：
* **左上角**：【生理体征区】心率和血氧卡片，使用带发光效果的红色字体，旁边有一个跳动的小心脏图标。
* **右上角**：【进度区】巨大的数字“15 / 20 次”，下方小字“第 2 组”。
* **正下方**：【AI OSD 弹窗】平时隐藏，当进入组间休息（Resting）或者动作错误时，底部升起一个宽幅的毛玻璃字幕条，同时左侧的 AI 渐变光球开始跳动，播放你通过前端控制的 TTS 语音，实现“教练在耳边指导”的错觉。
* **结算浮层**：训练结束，视频画面被深度模糊遮罩（`backdrop-blur-3xl`），中央弹出 5 个超大的 RPE 疲劳度 Emoji 选项卡片供用户点击。
